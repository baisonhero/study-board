# アプリケーションセキュリティ ツール分類（SAST/DAST/SCA/CNAPP）

> [!summary]
> AppSecの主要ツールカテゴリを「**いつ・どこで・何を**見るか」で整理。SDLCの工程ごとに役割分担があり、最近は CNAPP のように複数領域を1プラットフォームに統合する流れ。

## SAST（Static Application Security Testing）

ソースコードやバイトコードを **実行せずに** 解析する静的解析。

- **何を見つける**：SQLインジェクション、XSS、ハードコードされたシークレット、安全でないデシリアライゼーションなど、コード上のパターンで検出できる脆弱性
- **代表ツール**：Semgrep、SonarQube、GitHub CodeQL、Snyk Code、Checkmarx
- **位置**：IDE、pre-commit、PR時のCI
- **強み**：早い段階で回せて、開発者の手元に近いほど効く
- **弱み**：誤検知の多さ。ランタイム文脈なしには判断できない問題（例：認可ロジックの欠陥）は見えづらい

## DAST（Dynamic Application Security Testing）

**動いているアプリ** に対して外側からブラックボックステストを仕掛ける。

- **何を見つける**：認証バイパス、reflected/stored XSS、SQLi、サーバ設定ミス
- **代表ツール**：OWASP ZAP（OSS）、Burp Suite、Acunetix
- **位置**：staging環境、pre-prod、定期スキャン
- **強み**：本物の攻撃を再現するので false positive が少ない、言語非依存
- **弱み**：デプロイ済みでないと回せない、遅い、コードレベルのバグは見落としがち

## SCA（Software Composition Analysis）

自分のコードではなく **使っているサードパーティ依存** を調べる。

- **何を見つける**：依存ライブラリのCVE（Log4Shell、xz-utils backdoor等）、OSSライセンス問題
- **代表ツール**：Dependabot、Snyk Open Source、Trivy、Renovate、GitHub Advisory Database
- **位置**：CI/CD、リポジトリスキャン、SBOM生成
- **重要性**：サプライチェーン攻撃の増加でますます必須

## CNAPP（Cloud-Native Application Protection Platform）

2021年頃にGartnerが提唱した**統合カテゴリ**。クラウドネイティブな環境で「コードからクラウド、ランタイムまで」を1つのダッシュボードで見るためのプラットフォーム。

中身は複数のサブカテゴリの寄せ集め：

| サブカテゴリ | 何をやるか |
|---|---|
| **CSPM**（Cloud Security Posture Management）| AWS/GCP/Azureの設定ミス検出（S3が公開、SGが0.0.0.0/0開放等） |
| **CWPP**（Cloud Workload Protection Platform）| コンテナ・VM・サーバレスのランタイム保護 |
| **CIEM**（Cloud Infrastructure Entitlement Management）| 過剰なIAM権限の発見と最小化 |
| **IaC Scanning** | Terraform / CloudFormation の設定脆弱性 |
| **SAST/SCA統合** | 製品によってはコードレベル検査も同梱 |

- **代表ツール**：Wiz、Palo Alto Prisma Cloud、Aqua Security、Lacework、Orca Security、Microsoft Defender for Cloud
- **背景**：ツール乱立で疲弊した運用チームをまとめあげる方向の動き

## 「等」に入る関連カテゴリ

### IAST（Interactive AST）

SASTとDASTのハイブリッド。動作中のアプリにエージェントを仕込んで、リクエストとコードの両方の文脈で脆弱性を見つける。代表：Contrast Security。CIで回しづらく主流にはなり切れていない。

### RASP（Runtime Application Self-Protection）

アプリ内に組み込んで、攻撃が来た瞬間に**実行時に防御する**。検出ではなく「止める」側のアプローチ。

### Secret Scanning

コードベース・コミット履歴・ログ等からAPIキー・トークンの漏洩を見つける。SCAとは別カテゴリで扱われることが多い。

- 代表：GitHub Secret Scanning、TruffleHog、Gitleaks

### IaC Scanning

Terraform、CloudFormation、Kubernetesマニフェスト、Dockerfileの設定ミスを静的に見る。CNAPPに統合される流れ。

- 代表：Checkov、tfsec、KICS

### ASPM（Application Security Posture Management）

さらに新しい統合層。上記すべての結果を一元管理して「結局このアプリの優先度高い脆弱性は？」を出すメタダッシュボード。

## DevSecOpsシフトレフトでの位置関係

```
コード ─────── ビルド ─────── デプロイ ─────── ランタイム
SAST           SCA           DAST            CWPP / RASP
SCA            IaC Scan      IaC Scan        CSPM
Secret Scan    Container     Posture         CIEM
                Scan
                                    ←─── CNAPP（横串）─────→
```

## 最低限おさえる戦略

- **個人/小規模** → GitHub Dependabot（SCA）+ Secret Scanning + 軽量SAST（Semgrep / CodeQL）+ OWASP ZAP（DAST）の組み合わせから始めるのが王道
- **クラウドに本格デプロイし始めたら** CSPM が次に大事（設定ミスが事故の8割）
- **規模が出てから** CNAPP を検討。それまで個別ツールでOK

> [!tip] 学習の足がかり
> SAST/DAST/SCAは「**どの瞬間にどこを見るか**」で覚えると混ざらない。SAST=書いた瞬間にコード、SCA=ビルド時に依存、DAST=動かしてから外側、CNAPP=デプロイ後にクラウド全体。
