> Next.js + Supabase + Vercel スタックで個人開発・マネタイズを目指すAWS全冠保持者向けの、AppSec学習ロードマップ。Tier 1（実装に直結）→ Tier 2（運用と組織）→ Tier 3（広く長期）の順で進める。

## Tier 1 — 今すぐ手元のアプリに効く

実装に直結し、書いてるNext.js + Supabaseアプリの本番化に必要なもの。優先順：

- [ ] [[認証と認可]] — OAuth 2.0 / OIDC、JWT落とし穴、SupabaseのRLS、セッション管理
- [ ] [[Web脆弱性の実装防御]] — XSS/CSRF/SSRF/Open Redirect/CORS/CSP の実装側対策
- [ ] [[暗号の基礎]] — ハッシュ/暗号化/エンコーディング、パスワードハッシュ、TLSの保証範囲
- [ ] [[シークレット管理]] — .env運用、Vercel/Supabase secrets、AWS Secrets Manager、leak防止
- [ ] [[ソフトウェアサプライチェーン強化]] — Sigstore/cosign、SLSA、lockfile integrity、dependency confusion

> [!tip] Tier 1 の到達ゴール
> 「自分のSupabaseアプリで、RLSを正しく書き切って、本番デプロイできる状態」。これが完成すると、Tier 1 の知識が血肉になっている証拠。

## Tier 2 — 運用と組織のレベル

サービスを継続的に守る、チームや事業として運用するために必要：

- [ ] [[脅威モデリング]] — STRIDE、データフロー図、信頼境界、Threat Dragon
- [ ] [[インシデントレスポンス]] — 検知・対応・復旧・事後分析、ランブック、ログ保全
- [ ] [[OWASP API Security Top 10]] — API特有のリスク、BOLA、レート制限
- [ ] [[AWSセキュリティ実装]] — IAM最小権限、Inspector/GuardDuty/Security Hub、KMS実装
- [ ] [[セキュリティロギング設計]] — PII除外、構造化ログ、改ざん防止、保管期間（[[OTel]] と接続）

## Tier 3 — もう一段広く、長期的に効く

事業規模・社外説明・新興領域：

- [ ] [[コンプライアンスと法規制]] — GDPR、個人情報保護法、PCI DSS、SOC 2
- [ ] [[プロダクト開発における個人情報保護]] — 個人情報保護の実装ガイド、Privacy by Design、越境移転、削除/エクスポート機能
- [ ] [[LLMアプリケーションセキュリティ]] — OWASP LLM Top 10、プロンプトインジェクション、エージェント権限
- [ ] [[コンテナとKubernetesセキュリティ]] — イメージスキャン、Pod Security Standards、ネットワークポリシー
- [ ] [[ゼロトラストとネットワーク基礎]] — TLS、HSTS、DNS security、ゼロトラスト原則
- [ ] [[セキュリティ標準とフレームワーク]] — NIST CSF、ISO 27001、CIS Benchmarks、CVSS
- [ ] [[攻撃側視点とハンズオン学習]] — Juice Shop、HTB/THM、PortSwigger Academy、Burp/ZAP
