# AI駆動開発の変遷 2020-2026

2020年頃から2026年現在までのAI駆動開発（AI-driven development）の主要な出来事・ツール・パラダイムシフトを時系列でまとめる。

---

## Phase 1: コード補完時代（2021-2022）

### 主な出来事
- **2021年6月**: GitHub Copilot テクニカルプレビュー公開。OpenAI Codex（GPT-3ベース）を搭載し、VS Code上でインラインコード補完を提供
- **2021年10月**: Copilot が JetBrains マーケットプレイスにも展開
- **2022年6月**: GitHub Copilot 一般公開（GA）。個人向け月額$10
- **2022年**: Amazon CodeWhisperer プレビュー公開、Tabnine が GPT-3.5 対応

### 開発者の役割
- 開発者がコードを書く主体。AIは「賢いオートコンプリート」
- 提案されたコードを受け入れるか拒否するかは開発者が判断
- 調査によると75%以上の開発者が「反復的な作業の精神的負荷が減った」と回答

### パラダイムシフト
- 「AIがコードを書く」という概念が初めて大衆化
- ボイラープレートコード生成の自動化

---

## Phase 2: チャット型AI開発（2022-2023）

### 主な出来事
- **2022年11月**: ChatGPT 公開。コード生成・デバッグ・言語変換が可能と判明し、開発者が大量に利用開始
- **2023年3月**: GPT-4 リリース。コード生成精度が大幅向上
- **2023年3月**: GitHub Copilot Chat 発表（GPT-4ベース）
- **2023年**: 技術セクターで251社がChatGPTを業務導入。教育209社、ビジネスサービス98社が続く
- **2023年**: Codeium 創業（後のWindsurf）、コード補完プラグインとして開始

### 開発者の役割
- 自然言語で「やりたいこと」を説明し、AIが生成したコードをコピペする運用が一般化
- Stack Overflow の代替としてChatGPTを使う開発者が急増
- コードレビュー・バグ修正の相談相手としてAIを活用

### パラダイムシフト
- 「検索して調べる」から「AIに聞く」への行動変容
- 対話を通じた段階的なコード構築

---

## Phase 3: AIエディタ/IDE 時代（2023-2024）

### 主な出来事
- **2023年3月**: Cursor AI エディタ ローンチ（VS Code フォーク）
- **2023年後半**: Cline（VS Code拡張）登場。自律的にステップを踏み、エラーを自己修正する機能
- **2024年初頭**: Cursor が急速に普及。コードベース全体を理解した上での編集が可能に
- **2024年11月**: Windsurf 発表（Codeiumがリブランド）。Cascade機能でエージェント型自動コーディングを実現
- **2024年**: Aider（ターミナルベースのAIペアプログラマ）が注目を集める

### 開発者の役割
- AIがコードベース全体を理解し、複数ファイルにまたがる変更を提案
- 開発者は「何を作るか」を指示し、AIが「どう作るか」を担当する割合が増加
- ファイル作成・パッケージインストールまでAIが実行

### パラダイムシフト
- AIが単なる補完からコンテキスト理解型のペアプログラマへ進化
- IDE自体がAIファーストで再設計される流れ

---

## Phase 4: AIエージェント型開発（2024-2025）

### 主な出来事
- **2024年3月**: Devin（Cognition AI）発表。「世界初の完全自律型AIソフトウェアエンジニア」を謳う
- **2025年2月**: Andrej Karpathy が「vibe coding」（バイブコーディング）を提唱
- **2025年5月**: Claude Code リリース（Anthropic）。ターミナルネイティブのエージェント型コーディングツール
- **2025年5月**: OpenAI Codex エージェント リサーチプレビュー公開。クラウドベースで並列タスク実行
- **2025年5月**: Google Jules 発表。非同期エージェント型コーディング
- **2025年**: GitHub Copilot に Agent Mode 追加
- **2025年12月**: GPT-5.2-Codex リリース。コンテキストコンパクション導入

### 開発者の役割
- 「コードを書く人」から「AIエージェントに指示を出す人」へ
- タスクを自然言語で指示し、AIが計画→実装→テスト→PR作成まで一貫して実行
- 開発者はレビューと意思決定に集中

### パラダイムシフト
- 補完→対話→エージェントという進化の第3段階
- 「パスワードリセット機能を追加して」のような抽象的指示で実装が完了する世界
- SWE-bench でClaude Code が80.9%を達成

---

## Phase 5: 2026年現在の最新動向

### 自律型・マルチエージェント
- **2026年2月**: 主要ツールが同時期にマルチエージェント機能をリリース。複数エージェントの同時実行が標準に
- **2026年2月**: Karpathy が「vibe coding」は素人向けと再定義し、プロ向けには「agentic engineering」を提唱
- **2026年3月**: Cursor Composer 2 リリース。マルチファイル編集に特化した独自フロンティアモデル
- **GPT-5.3-Codex**: OpenAI 最新のエージェント型コーディングモデル
- **MCP（Model Context Protocol）**: Anthropic発のプロトコルが事実上の標準に。60,000以上のOSSプロジェクトが採用

### 市場シェア（2026年4月時点）
- GitHub Copilot: 認知度76%、業務利用29%（成長は停滞）
- Claude Code: 業務利用18%、「最も好きなツール」46%で1位。GitHub Stars 51,000+
- Cursor: 「最も好きなツール」19%
- Devin: 最も自律性の高いクラウドエージェント

### バイブコーディングからエージェンティックエンジニアリングへ
- PEVループ（Plan → Execute → Verify）がコアワークフロー
- 専門化されたエージェントが定義された役割で協調動作
- 人間はアーキテクトとしてエージェント群を設計・監督
- Stripeでは社内AIエージェントが週1,000以上のPRを自動生成・マージ

### 一般的な使い分け
- IDE系エージェント（Cursor等）: 日常の開発作業
- ターミナル系エージェント（Claude Code等）: 難易度の高い問題
- クラウド系エージェント（Devin等）: バックグラウンドでの非同期タスク

## 主要プレイヤーの動き

| プレイヤー | 主なプロダクト | 戦略 |
|---|---|---|
| **GitHub/Microsoft** | Copilot → Agent Mode | 最大の認知度。VS Code統合。成長は停滞 |
| **Anthropic** | Claude Code | ターミナルネイティブ。MCP標準化。開発者満足度1位 |
| **OpenAI** | Codex CLI → Codex Agent | クラウドベース並列実行。GPT-5系列で進化 |
| **Google** | Gemini Code Assist / Jules | Gemini 2.5 Pro搭載。非同期エージェント |
| **Anysphere** | Cursor | AI専用IDE。独自モデル（Composer 2）開発 |
| **Cognition AI** | Devin | 完全自律型。サンドボックス環境で独立動作 |
| **Codeium→Windsurf** | Windsurf | Cascade機能。自動化重視の哲学 |

## 開発者の役割の変遷まとめ

```
2021: コードを書く人（AIは補完ツール）
  ↓
2022-23: AIに質問しながらコードを書く人（AIは相談相手）
  ↓
2024: AIと協働してコードを書く人（AIはペアプログラマ）
  ↓
2025: AIに指示を出してレビューする人（AIはジュニア開発者）
  ↓
2026: エージェント群を設計・監督するアーキテクト（AIは開発チーム）
```
