# MOC Comms

> [!summary]
> メール・Slack・ミーティング・外部発信などの **コミュニケーション記録の入口** 。やり取りの発生源（Gmail / Slack / Notion / X 等）に分散しがちな履歴を、Vault 上の一次情報・要約・意思決定に紐付ける。

## なぜこのMOCがあるか

外部とのやり取りは技術ノートと違い、後から発掘しにくい。「このトピック、誰と何月にどういう話をしたか」「あの結論はどのスレッドが起点だったか」を追跡可能にしておくと、契約・採用・コラボのスピードが上がる。日次ログ（`_daily/`）の「コミュ系」項目はここに集約する。

## 運用ルール

- 1スレッド/1ミーティング = 1ノートを基本とする
- ファイル名: `YYYY-MM-DD <相手・トピック>` で時系列に並ぶように
- 必ず以下を残す：
  - 日時 / 相手 / 媒体（Gmail / Slack / Zoom / 対面）
  - 要点（5行以内）
  - 決定事項 → `#decision` タグ
  - 次アクション → ToDo として `_daily/` に転記
  - 出典リンク（メール permalink / Slack archive link / Notion ページ）
- 機微情報は要約のみ。生のメール本文をベタ貼りしない
