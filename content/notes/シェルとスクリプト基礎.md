# シェルとスクリプト基礎

> [!summary]
> Bash と テキスト処理（awk / sed / grep）。**インフラ業務の8割はテキスト整形と簡単な自動化**。手早く書けるシェルスクリプトと、Pipeline での組合せの定番イディオムを押さえる。

## Bash の基本

### 変数

```bash
NAME=alice                  # 代入（=の前後にスペース禁止）
echo $NAME
echo "${NAME}"              # 推奨表記
echo "${NAME:-default}"     # 未定義ならデフォルト

# 環境変数として export
export PATH="$HOME/bin:$PATH"

# 算術
COUNT=$((10 + 20))
```

### 文字列とクオート

```bash
# シングルクオート：変数展開しない
echo '$NAME'                # → $NAME

# ダブルクオート：変数展開する
echo "$NAME"                # → alice

# バッククオート / $()：コマンド実行結果を埋め込む
DATE=$(date +%Y-%m-%d)
LIST=`ls`                   # 古い書き方、$()推奨
```

### 条件分岐

```bash
if [ "$NAME" = "alice" ]; then
    echo "Hi alice"
elif [ -z "$NAME" ]; then
    echo "Name is empty"
else
    echo "Hello $NAME"
fi

# 数値比較
if [ "$COUNT" -gt 10 ]; then
    echo "many"
fi

# ファイル存在
if [ -f /etc/passwd ]; then
    echo "exists"
fi

# &&、|| の短絡評価
mkdir mydir && cd mydir
test -f file.txt || echo "no file"
```

### ループ

```bash
# for
for i in 1 2 3 4 5; do
    echo $i
done

for f in *.log; do
    echo "Processing $f"
done

# 数字シーケンス
for i in {1..10}; do echo $i; done
for i in $(seq 1 10); do echo $i; done

# while
i=0
while [ $i -lt 10 ]; do
    echo $i
    i=$((i + 1))
done

# 行を読み込む
while read line; do
    echo "Line: $line"
done < file.txt
```

### 関数

```bash
greet() {
    local name=$1
    echo "Hello, $name"
}

greet alice
greet "$@"                  # 全引数を渡す
```

### exit code

```bash
command
echo $?                     # 直前コマンドの終了コード（0=成功）

set -e                      # エラーで即終了（スクリプトの推奨設定）
set -u                      # 未定義変数でエラー
set -o pipefail             # パイプラインの途中エラーを検知

# 安全なスクリプト先頭の定番
#!/bin/bash
set -euo pipefail
```

## テキスト処理パイプライン

### grep（検索）

```bash
grep "pattern" file
grep -i "pattern" file              # 大文字小文字無視
grep -v "pattern" file              # マッチしない行
grep -r "pattern" dir/              # 再帰
grep -n "pattern" file              # 行番号付き
grep -A 3 -B 1 "pattern" file       # マッチ前後
grep -E "regex|alternative" file    # 拡張正規表現
grep -P "perl-regex" file           # Perl互換regex
```

### sed（置換）

```bash
# 置換
sed 's/old/new/' file               # 各行最初のみ
sed 's/old/new/g' file              # 全置換

# 直接書き換え
sed -i 's/old/new/g' file
sed -i.bak 's/old/new/g' file       # .bakバックアップを作る

# 特定行を削除
sed '/pattern/d' file               # patternを含む行
sed '5d' file                       # 5行目
sed '5,10d' file                    # 5-10行目
```

### awk（フィールド処理）

```bash
# 第2フィールドを表示
awk '{print $2}' file

# CSV風（カンマ区切り）
awk -F',' '{print $1, $3}' file

# 条件付き
awk '$3 > 100 {print $1, $3}' file

# 集計
awk '{sum += $1} END {print sum}' numbers.txt
awk '{count[$1]++} END {for (k in count) print k, count[k]}' file

# 行番号
awk 'NR == 5 {print}' file          # 5行目だけ
awk 'NR > 1' file                   # ヘッダースキップ
```

### sort、uniq

```bash
sort file
sort -n file                        # 数値ソート
sort -r file                        # 逆順
sort -u file                        # uniqも同時
sort -k 2 file                      # 2列目でソート
sort -k 2 -t ',' file               # CSVで2列目

uniq file                           # 連続重複排除（要sort）
uniq -c file                        # カウント付き
sort file | uniq -c | sort -rn      # 頻度順
```

### head / tail

```bash
head -n 10 file
tail -n 10 file
tail -f file                        # リアルタイム追跡
tail -F file                        # ローテーションも追従

# 末尾以外
head -n -10 file                    # 最後の10行を除く
tail -n +10 file                    # 10行目から末尾
```

### cut

```bash
cut -d ',' -f 1,3 file              # CSV、1列目と3列目
cut -c 1-10 file                    # 各行の1〜10文字目
```

### tr（文字変換）

```bash
echo "hello" | tr 'a-z' 'A-Z'       # 大文字化
tr -d '\r' < input > output         # CR削除（dos2unix代用）
tr -s ' '                           # 連続空白を1つに
```

### xargs

```bash
# 各行を引数として渡す
ls *.log | xargs wc -l

# 並列実行
ls *.log | xargs -P 4 -I {} gzip {}

# null区切り（スペース含むファイル名対応）
find . -name "*.log" -print0 | xargs -0 wc -l
```

### find

```bash
# 名前で
find . -name "*.log"
find . -iname "*.LOG"               # 大小無視

# タイプで
find . -type f                      # ファイル
find . -type d                      # ディレクトリ

# 時間で
find . -mtime -7                    # 7日以内に更新
find . -mtime +30                   # 30日以上前
find . -mmin -60                    # 60分以内

# サイズで
find . -size +100M

# アクション
find . -name "*.log" -delete
find . -name "*.tmp" -exec rm {} \;
find . -name "*.log" -exec gzip {} \;
```

## 実用パイプラインの例

### アクセスログでHTTPステータスコード集計

```bash
awk '{print $9}' access.log | sort | uniq -c | sort -rn
```

### 上位10のIPアドレス

```bash
awk '{print $1}' access.log | sort | uniq -c | sort -rn | head -10
```

### ディスクの大きいディレクトリを見つける

```bash
du -sh /* 2>/dev/null | sort -h | tail -20
```

### 特定プロセスのメモリ使用量を継続記録

```bash
while true; do
    ps -p $(pgrep -f myapp) -o pid,rss,cmd
    sleep 10
done >> mem.log
```

### CSVから特定カラムだけ抜き出して別CSV

```bash
awk -F',' 'NR > 1 {print $1","$3","$5}' input.csv > output.csv
```

### 対話式でログを検索

```bash
tail -f /var/log/nginx/access.log | grep --line-buffered "ERROR"
```

`--line-buffered` 重要、これがないと出力がバッファリングされて遅延する。

## エラーハンドリング

### スクリプトの安全なヘッダ

```bash
#!/usr/bin/env bash
set -euo pipefail
IFS=$'\n\t'                         # 安全なIFS（スペースを含むファイル名対策）

# 終了時にクリーンアップ
trap 'rm -f /tmp/myscript.tmp' EXIT
```

### ログとデバッグ

```bash
# デバッグ用に各行を表示
set -x

# 関数化
log_info()  { echo "[INFO]  $*" >&2; }
log_error() { echo "[ERROR] $*" >&2; exit 1; }

[ -f "$FILE" ] || log_error "File not found: $FILE"
```

## 引数解析（スクリプトの引数）

簡易：

```bash
NAME=${1:?usage: $0 <name>}
COUNT=${2:-10}                      # デフォルト10
```

`getopts`（POSIX）：

```bash
while getopts "n:c:vh" opt; do
    case $opt in
        n) NAME=$OPTARG ;;
        c) COUNT=$OPTARG ;;
        v) VERBOSE=1 ;;
        h) echo "Usage: $0 -n NAME [-c COUNT]"; exit 0 ;;
        *) exit 1 ;;
    esac
done
```

複雑なら `argparse` を持つ言語（Python等）に切り替える判断も。

## ShellCheck（推奨）

```bash
shellcheck script.sh
```

シェルスクリプトの linter。**書いたら必ず通す**習慣を。よくあるバグ（クオート忘れ、`==` vs `=` など）を検出。

## いつシェルでなく他の言語に切り替えるか

- **複雑なデータ構造**（JSON操作多い）→ Python
- **150行を超える** → Python
- **テストを書きたい** → Python（pytest）
- **複雑なエラーハンドリング** → Python or Go
- **配列・辞書を真面目に使う** → Python

シェルは**接着剤**として優秀、ロジック表現は他言語に任せる。

## jq（JSON処理）

シェル時代のJSON必須ツール：

```bash
# .field を抽出
echo '{"name":"alice","age":30}' | jq '.name'

# 配列フィルタ
echo '[1,2,3,4]' | jq '.[] | select(. > 2)'

# 整形
cat data.json | jq '.'

# CSVに変換
jq -r '. | [.name, .age] | @csv' data.json
```

`yq` は YAML/TOML/XML対応の同等ツール。
