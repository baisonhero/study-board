# 攻撃側視点とハンズオン学習

> [!summary]
> 守るためには攻める側を知る必要がある。**HackTheBox / TryHackMe / PortSwigger Web Security Academy**（無料・最高品質）、Bug Bounty platforms、Burp Suite、OWASP ZAP、Juice Shop。読書だけでは身につかないので、手を動かす学習法。

## なぜ攻撃側を学ぶのか

- 守備側だけだと「どこが穴か」の感覚が鈍る
- 自分のサービスを脅威モデリングする時の解像度が違う
- セキュリティ修正のレビューで的外れな指摘をしなくなる
- 報告された脆弱性の深刻度を**自分で評価**できる
- インシデントの**仮説**を立てる時の引き出し

## 学習プラットフォーム

### 無料・初心者〜中級

#### PortSwigger Web Security Academy

- **無料・登録だけでOK**
- Web脆弱性に特化、**業界最高品質**
- カテゴリ別ラボ：SQLi、XSS、CSRF、SSRF、Authentication、Access Control、API、JWT、GraphQL等
- 解説 → ラボ実践 → 解答ヒント の流れ
- Burp Suite Community Edition と組み合わせる

最初の1か月の取り組み先として強く推奨。AppSec実装者がここを通過してると話の解像度が違う。

#### TryHackMe

- 教育寄り、丁寧なルームベース
- **無料枠あり**、有料サブスクで全アクセス
- 「Pre Security」「Complete Beginner」等の学習パス
- AD（Active Directory）攻撃、Linux特権昇格、ネットワーク偵察まで網羅
- ゲーム性があって続けやすい

#### HackTheBox

- 中級〜上級、ガチ寄り
- マシンを攻略する形式（root取るまで）
- Academy（有料）が体系的
- 履歴書やSNSで実績を出せる

### 攻撃ハンズオン用の意図的に脆弱なアプリ

#### OWASP Juice Shop

- ローカルで立ち上げて自分で攻撃する練習台
- Node.js、現代的なSPAで、現実のWebアプリに近い
- 100以上のチャレンジ、難易度別

```bash
docker run --rm -p 3000:3000 bkimminich/juice-shop
```

これが一番手軽。

#### DVWA、bWAPP

- 古典的、PHP、レガシー脆弱性の網羅性が高い
- レガシー系を学ぶには有用

#### WebGoat

- OWASPのJava製、設計思想を学べる

## 攻撃ツール

### Burp Suite

- Webアプリ脆弱性診断のデファクト
- **Community Edition** は無料、Repeater / Decoder / Comparer / Proxyが使える
- **Professional**（年$449）で Scanner、Intruder（強化版）、保存/拡張等
- Extender でカスタムプラグイン（[[OWASP]] のSemgrep連携、ChatGPT連携等多数）

### OWASP ZAP

- OSSのDAST、Burpの代替
- 自動スキャンが強力、CIに組み込みやすい
- API テストモードあり

### sqlmap

- SQL Injection 自動エクスプロイト
- 練習場での検出 → 自動exploit → DB dumpの流れを体験できる

### nmap、masscan

- ポートスキャン、サービス検出
- 学習環境を超えて使うときは**所有許可**を必ず確認（不正アクセス禁止法）

### Metasploit

- 大規模なエクスプロイトフレームワーク
- HackTheBox等の演習で使う

## Bug Bounty

実戦の世界に足を踏み入れたい場合：

### プラットフォーム

- **HackerOne** — 最大手、Stripe、GitHub、Uberなどがプログラム公開
- **Bugcrowd** — 同等のもう一つの大手
- **Intigriti** — 欧州系
- **Synack** — 招待制、レベル高い
- **YesWeHack** — 欧州系

### 始め方

1. プラットフォーム登録、scope読み込み
2. 公開プログラム（VDP）から試す（報酬は出ないが安全に経験）
3. 簡単な脆弱性（XSS、Open Redirect、Sub Takeover）から
4. レポート作成スキル（再現手順、影響、修正案）が地味に重要
5. デュプリケート（既報）が多いのでめげない

> [!warning] 重要：scope外を絶対に攻撃しない
> 不正アクセス禁止法違反。プログラムに参加しても、**明示されたscope内**だけに留める。範囲外発見時は「うっかり触った」段階で止めて報告に切り替える。

## レッドチーム / Blue Team / Purple Team

- **Red Team** — 攻撃側、ペネトレーションテスター
- **Blue Team** — 防御側、SOCアナリスト、IRハンドラー
- **Purple Team** — 両方を行き来する協働

個人開発者が学ぶならまず **Purple チーム志向**が現実的。攻撃側の引き出しを増やしながら、自分のサービス防御に活かす。

## 学習の進め方（推奨順）

1. **PortSwigger Web Security Academy** を1〜2か月で主要トピック踏破
2. 並行して **OWASP Juice Shop** を docker で立ち上げてチャレンジ
3. **Burp Suite Community** を実プロジェクトの開発環境で実際に通信を見る練習
4. **TryHackMe** の "Complete Beginner" パスで Linux / ネットワーク基礎
5. 自分のサービスに **OWASP ZAP** を回して自動スキャン
6. （関心があれば）HackerOne の VDP を1件レポートしてみる

## 倫理的・法的ライン

- **不正アクセス禁止法**（日本）— 許可なきアクセスは犯罪
- 自分が所有するサーバ・許可された演習場・bug bounty scope内 のみ
- 「**脆弱性を発見しても報告以上のことはしない**」
- 公開情報の収集（OSINT）でも、相手によっては問題になる場合がある
